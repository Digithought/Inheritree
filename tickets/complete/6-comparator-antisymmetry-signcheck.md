## What shipped

`compareKeys` in `src/b-tree.ts` now runs a **sign-based** antisymmetry self-check on user comparators:

```ts
if (Math.sign(this.compare(b, a)) !== -Math.sign(result)) {
    throw new InconsistentComparatorError();
}
```

Replaces the old identical-value test (`result !== 0 && result === this.compare(b, a)`). One shared code path, so both the default bounded-sample mode (first `SampleCheckCount` = 32 comparisons) and `{ checkComparator: true }` (exhaustive) get the stronger check with no added branching.

Now caught that the old check missed:
- Same-sign, different-magnitude results (`1` vs `2`) — the broken-subtraction shape.
- Zero-asymmetry (`compare(a,b) === 1`, `compare(b,a) === 0`).
- Bonus: NaN-returning comparators (`Math.sign(NaN)` is NaN → mismatch → throws).
